# Support

Use GitHub Discussions, when enabled, for usage questions and GitHub Issues for reproducible bugs or focused feature requests. Search existing reports first and use the provided templates.

Before posting, remove hostnames, IP addresses, usernames, commands containing secrets, private paths, credentials, keys, terminal captures, and unredacted diagnostics. Security vulnerabilities must be reported privately as described in [SECURITY.md](SECURITY.md).

This community project provides support on a best-effort basis. The latest release is the supported version; maintainers may ask reporters to reproduce an issue there before investigating.
