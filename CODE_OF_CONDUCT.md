# Code of conduct

OmniTerm is committed to a welcoming, professional, and harassment-free project community. Be respectful, discuss technical decisions in good faith, accept constructive feedback, and make space for people with different backgrounds and levels of experience.

Harassment, threats, discriminatory language, sexualized attention, deliberate intimidation, publication of another person's private information, and sustained disruption are not acceptable in project spaces.

Maintainers may edit or remove contributions, comments, or other participation that violates these standards. They may issue a warning, temporary restriction, or permanent ban in proportion to the behavior and its impact. Enforcement decisions should protect reporters' privacy and avoid retaliation.

Report conduct concerns privately through the contact methods on the repository owner's GitHub profile. Security vulnerabilities belong in GitHub's private vulnerability reporting flow described in [SECURITY.md](SECURITY.md), not in a public issue.

This policy applies in repository discussions, issue trackers, reviews, project chat, and public representation of the project.
